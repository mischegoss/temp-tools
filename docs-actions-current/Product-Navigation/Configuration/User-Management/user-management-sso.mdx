You can integrate VAR::PRODUCT_FULL with third-party SAML Identity Providers (IdPs) to log in to VAR::PRODUCT. Examples include services such as Azure, Ping Identity, or Okta, as well as any custom SAML service.

Under **Configuration > User Management > SSO**, you can view the SAML SSO configuration of the VAR::PRODUCT instance. The accordion item displays the following information:

| Column | Description                              |
|--------|------------------------------------------|
| SAML Metadata URL    | The SAML URL provided by your IdP        |
| App Unique ID | The SAML Entity ID generated by your IdP |

From here, you can [edit an existing SSO configuration](../Single-Sign-On/configuring-resolve-actions-for-saml-sso.mdx#editing-the-sso-configuration), as well as [delete it](../Single-Sign-On/configuring-resolve-actions-for-saml-sso.mdx#deleting-the-sso-configuration).

[Creating a new SSO configuration](../Single-Sign-On/configuring-resolve-actions-for-saml-sso.mdx#enabling-sso) through the **+** icon in the top right of the **SSO** tab is only active when no configuration is present. Please note that you can only integrate with a single SSO provider at any given time. If you wish to switch to a different SSO provider, you will need to delete the existing configuration and create a new one.
