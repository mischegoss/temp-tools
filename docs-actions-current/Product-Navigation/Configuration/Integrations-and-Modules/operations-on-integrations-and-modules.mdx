## Adding an Integration

Click the plus icon in the top right corner of the **Integrations** list.  
   The Integrations Configuration module appears.

#### Part One: Setup Connection

1. In the **Name** field, enter a name for the new integration.
2. In the **Description** field, enter a description.
3. From the **Type** field, select the integration type.
4. (Optional) In **Mode**, select where you want the integration to run:
   * **Cloud**—The integration will run in your VAR::PRODUCT_FULL cloud instance. This option is suitable for integration with services running in the cloud or on-premises services accessible from the cloud.
   * **Remote**—In hybrid clouds, the integration will run on the on-premises server where you installed the VAR::PRODUCT [hybrid components](../../../Getting-Started/Setting-Up-Hybrid-Components/setting-up-hybrid-components.mdx). This option is suitable for integration with services that run on-premises in a hybrid environment and are inaccessible from the cloud.
5. In **Log Level**, select the level of logging to enable.
6. Check **Monitor** to determine that VAR::PRODUCT will monitor the integration. When you select this option, a new incident is created when the integration is down.
7. Click **Save & Continue.**

:::note
**Log Level 1:** Writes service status and errors during the component's activity.

**Log Level 2:** Writes service status, errors, warnings, and unusual events during the component's activity.

**Log Level 3:** Writes all events and errors during the component’s activity. *Use only for advanced debugging. This option may result in writing very large logs which subsequently may result in slowing down the server.*
:::

#### Part Two: Connection Parameters

1. In the **Access Key** field, enter the access key.
2. In the **Secret Key** field, enter the secret key.
3. Click **Save & Continue**. 

#### Part Three: Permissions
1. In **Permissions**, select *Enable Permissions* to set permissions by role.
   See [Access Control on Integrations and Modules](./access-control-on-integrations-and-modules.mdx) for details and limitations.
2. In the **Type** column, select *User* or *Group*.
3. In the **Name** column, choose the User or Group name to apply permissions.
4. Select **Read** or **Write** permissions. 
5. To add additional users or groups, select **+ Add Permission.** 
6. Click **Save & Continue.**

## Editing an Integration

Once complete, your integration will appear in the *Integrations* list. Clicking the Integration name will open a drawer, allowing edits to the Integration and an option to **Test Connection**. Updates to the Connection Parameters and Permissions can be made in this drawer.

Select the three dots at the top right of the drawer to view the following action icons:
