## Activity Description

Synchronizes Azure AD users/groups and their data with the VAR::PRODUCT_FULL recipients list. The activity imports Groups' and Users' information from Azure AD into Actions.

## Output

Success / Failure.

## Settings


* **Username** – Username of an administrator account that will be accessing the Azure portal to sync users into
* **Password** – The password associated with the account used above.
* **App Client ID** – The App Client ID associated with the Azure AD where the users reside.
* **AD Users** – The list of AD users/groups to synchronize. This can be static (the object ID for a single user and/or group) or a reference to a memory table created for this purpose. An example memory table is shown below.  
   Example: `%AzureID%`
* **Update Existing** – By default, if a user or a group already exists in Actions (created manually before synchronization), they will not be updated. When this is checked, existing users are updated.
* **Recursive Groups** – When a group is synchronized you may decide whether to synchronize the data recursively or not. When synchronizing the data recursively, sub-groups and user data are also synchronized.

:::note
The fields that will be synced from Azure AD to Resolve Actions are: Username, First Name, Last Name, Email, and Mobile Phone.

Username, First Name, and Last Name are required. Email and Mobile Phone are highly recommended to contact recipients once they've been synced into Resolve Actions.
:::
