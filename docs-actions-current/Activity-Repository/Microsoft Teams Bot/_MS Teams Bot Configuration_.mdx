# Introduction

Creating a Bot with MS Teams is straightforward with credential-free authentication using a Hash-Based Message Authentication Code (HMAC). This method  generates a secure key to receive messages through Teams.

## Creating a Secret Key

1. Click the hamburger menu at the top right and navigate to **Configuration > API Access**.
2. Click the + sign in the *Secret Keys* menu and fill in the fields:
   *  **Name:** Create a unique name to identify its purpose and the Teams channel.
   *  **Type:** Select HMAC.
   *  **Value:** This value will be generated by teams.


# Create the HMAC Value

To generate the webhook and HMAC value in Microsoft Teams, use the following steps:
1. Select **Teams** from the left of the pane.
2. Find the team to create an Outgoing Webhook and select the three dots to the right of the name.
3. Select **Manage team.**
4. Select **Apps** on the channel page.
5. Select **Create an Outgoing Webhook.**
6. Fill in the following details:
   * **Name:** A name for the Bot. *Note*: Do not use spaces in the name.
   * **Callback URL:** The event URL in Resolve Actions. 
   * **Description:** A description of the bots function.
   * **Upload Image:** Select an icon for Teams. 
7. Press **create** and a dialogue box will appear with the HMAC key. 
8. Copy the HMAC key and paste it in the **Value** box in the Resolve Actions secret key configuration. 

:::caution
The HMAC key is only visible once. Please ensure you copy it when clicking **Create.**
:::

:::note
For best results, use the MS Teams Web Client instead of the Desktop Client.
:::

## Using the Teams Bot in a Workflow
To use the message sent from Teams into Resolve, you will need to create a GlobalVariable called **RawData**. To do this:
1. Navigate to the main menu using the hamburger icon and select **Repository > General**.
2. In the **Global Variables** menu, select the + at the right.
3. In the **Name** field, add *RawData*.
4. In the **Mode** field, select *Read-Only Mode*.
5. In the **Type** field, select *Text*. 

This variable will be populated with the JSON content from the Teams Bot, which can be used in a Workflow. For optimal results, use the JSON data in the Data Mapper activity. This will allow you to split the JSON content into individual variables and use specific pieces of information throughout the workflow.



## Using the MS Teams Bot
The Bot can be used in an MS Teams channel by referring to it with the @ function and asking it a question. 

By default, the Bot will respond with the same question until you define a response or reply via the Resolve Actions platform.
