## Activity Description

Checks the availability of a URL based on a return code or an existence of a specific phrase.

## Output

Success/Failure

## Settings


* **URL** – The website's address (e.g. `http://www.example.com/`). The address must begin with `HTTP://`.
* **User/Password** – If required, the username of an account that has read-write access to the host.
* **User Name** – The username by which the selected proxy server is accessed (optional, usually not required).
* **Password** – The password by which the selected proxy server is accessed (optional, usually not required).
* **Certificate** – If required certificate to authenticate, browse the certificate from the file system.
* **Certificate Password** – If required, type the certificate password.
* **Proxy** – The address of the proxy server that must be passed to reach the website (e.g., 192.168.10.10).
* **User Name** – A username authorized to access the proxy server (e.g., admin).
* **Password** – The password of the user authorized to access the proxy server.
* **Bypass proxy server for the following addresses** – Used for exceptional addresses not required to pass through the proxy servers.
* **Check By (Phrase/Error Code)** – Select the method of checking the URL's availability (either Phrase or Error Code).
* **Search Phrase** – The specific phrase searched to check the website's availability (not required if checking by error code).
