## Activity Description

Returns the open duration of an Incident (in minutes). This can be used, for example, to trigger a workflow only when an incident is open for more than a specified number of minutes.

## Output

The number of minutes for which an incident is open.

## Settings


* **Use Current Incident/Use Another Incident** – Determines whether to check the duration of the incident that triggered the workflow, or another incident.
* **Type: Device/Service** – Determines whether the incident is ascribed to a Device or a Service.
* **Name** – The name of the constant/dynamic incident's Device/Service (if you selected a constant incident, the list changes according to Type selection).  
   Example: `%DeviceName%`
* **Classification** – The classification of the incident's Device/Service (if you selected using a constant incident, the list changes according to Type selection).  
   Example: `%AlertType%`
